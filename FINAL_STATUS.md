Public Sale / Access Logic Status
DMD is currently documented with public sale mode as the relevant active access structure.
The contract may contain legacy whitelist/access logic from earlier presale structures. This logic remains documented for transparency, but it should not be confused with the current public sale access model.
Current documented buy limits:
Minimum buy amount: `0.1 SOL`
Maximum buy amount: `100 SOL`
Public sale access: documented as active according to the current project state
Sell status: depends on the active on-chain system state, treasury configuration and protocol rules
Important distinction:
The minimum buy amount defines how much SOL a user can contribute in a buy transaction.
Legacy whitelist balance checks belong to earlier presale/access logic.
Public sale mode, buy limits, whitelist logic, claim functions and sell status are separate mechanisms.
Users are responsible for reviewing the current Investor App state, public references and on-chain system status before interacting with DMD.
