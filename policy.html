Sale Access Notice

DMD may contain legacy whitelist and access logic from earlier presale structures. Current public access depends on the active on-chain system state.

Public sale mode, buy limits, whitelist logic, claim functions and sell status are separate mechanisms. They should not be understood as one single condition.

  • The minimum buy amount defines the SOL contribution required for a buy transaction.
  • Legacy whitelist logic belongs to an earlier presale/access structure and remains documented for transparency.
  • Public sale access depends on the active system state and current project configuration.
  • Sell status may depend on on-chain state, treasury configuration, vault balance and protocol rules.

Users are responsible for reviewing the current public references, Investor App state and on-chain system status before interacting with DMD.
